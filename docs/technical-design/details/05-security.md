# 安全详细设计

> 本文是安全专项详细设计。整体技术路径见 `../00-technical-blueprint.md`。

## 1. 敏感信息原则

CPClaw 不得在代码、文档、日志、Git 提交、LLM Prompt、模型调用审计、工具调用审计和附件解析记录中保存明文敏感信息。

敏感信息包括：

- 云枢密码。
- GitHub 密码或 Token。
- 模型 API Key。
- Token、Cookie、Session ID。
- 数据库密码。
- 检索中间件密码。
- 附件中的敏感业务原文。
- Authorization Header。

## 2. 凭据存储

- 云枢密码通过设置页输入。
- 模型 API Key 通过设置页输入。
- 检索中间件密码通过设置页输入。
- 后端使用 `CPC_ENCRYPTION_KEY` 进行 AES-GCM 加密。
- MySQL 中仅保存密文、IV、Auth Tag。
- 前端不回显密码或 API Key。
- API 不返回密文内容。
- 凭据只在运行时解密并注入对应请求，不写日志。

## 3. 环境变量

推荐通过 `.env` 或部署平台 Secret 提供：

- `DATABASE_URL`
- `DATABASE_USERNAME`
- `DATABASE_PASSWORD`
- `CPC_ENCRYPTION_KEY`

`.env`、本地配置、密钥文件和日志必须加入 `.gitignore`。

## 4. 日志脱敏

以下字段必须脱敏：

- `password`
- `token`
- `cookie`
- `authorization`
- `api_key`
- `secret`
- `credential`
- `session`
- `key`

脱敏范围包括：

- 后端应用日志。
- HTTP 请求/响应日志。
- 工具调用审计。
- 模型调用摘要。
- 浏览器自动化截图说明。
- 附件解析错误。

## 5. Prompt 与模型调用安全

- 不将凭据放入 LLM Prompt。
- 不将 Token、Cookie、Session ID 放入 Prompt。
- 附件敏感原文不进入长期记忆。
- Prompt 中只放执行所需的最小化业务上下文。
- 工具只接收必要参数。
- 高风险工具执行前由后端业务逻辑拦截确认，不只依赖模型自觉。
- Agent 输出不得包含内部 Token、Cookie、API Key 或密文。
- 模型调用审计只保存脱敏摘要、模型配置 ID 和调用状态。

## 6. 操作确认

以下操作必须用户确认：

- 删除。
- 批量修改。
- 新增或修改关键业务数据。
- 流程提交。
- 触发业务 Action。
- 大批量导出。
- 附件上传到云枢。
- 浏览器自动化写入操作。

确认记录需要包含：

- 操作摘要。
- 影响对象。
- 修改前后差异。
- 附件列表。
- 风险等级。
- 过期时间。

用户修改输入后，原确认失效。

## 7. 云枢权限

- 不绕过云枢现有权限体系。
- 使用配置账号或当前用户上下文访问云枢。
- 查询和操作结果以云枢权限返回为准。
- “我的数据”必须基于云枢当前用户身份和权限规则。
- 管理员同步元数据不代表业务用户拥有运行态操作权限。

## 8. 附件安全

- 附件类型白名单。
- 附件大小限制。
- 文件名规范化，避免路径穿越。
- 存储路径不可直接暴露。
- 下载和解析结果按会话权限访问。
- 附件解析结果落库前脱敏。
- 上传云枢前必须展示字段映射和附件确认。

## 9. 浏览器自动化安全

- 浏览器状态目录不得入库。
- 截图和 DOM 摘要需要脱敏。
- 浏览器兜底不能绕过确认机制。
- 浏览器执行步骤需要审计。
- 失败截图中可能包含敏感信息，保存和展示前需脱敏或受控访问。

## 10. GitHub 安全

- 不使用账号密码推送代码。
- 推荐使用 GitHub CLI 登录或具备最小权限的 PAT。
- 禁止将 PAT 写入仓库文件。
- 文档、配置样例、提交记录中不得包含明文密码、Token、Cookie、API Key。
- 文档必须经用户确认后才允许 commit 或 push。

## 11. 可测试控制项

- `.env` 不入库检查。
- Git diff 敏感词扫描。
- 设置接口不返回密码、API Key 或密文内容。
- 日志脱敏单元测试。
- 工具调用脱敏单元测试。
- 模型调用摘要脱敏测试。
- 附件路径穿越测试。
- 高风险操作未确认不得执行测试。
- 确认过期后不得执行测试。
