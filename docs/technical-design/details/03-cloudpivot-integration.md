# 云枢集成详细设计

> 本文是云枢集成专项详细设计。整体技术路径见 `../00-technical-blueprint.md`。

## 1. 集成目标

CPClaw 需要通过登录后的用户身份访问云枢设计态和运行态，理解应用架构并执行用户授权的业务操作。

关键原则：

- 云枢元数据先同步到本地 MySQL。
- Agent 检索和能力匹配只查询本地 Metadata Index，不实时连接云枢。
- 云枢实时连接只用于连接测试、元数据同步和经确认的运行态操作。
- 不绕过云枢权限体系。
- 业务问答必须基于真实云枢元数据和真实运行态数据；本地 fallback 数据只能用于连接/流程自测，不能作为业务答案。

## 2. 入口配置

系统设置中配置：

- 云枢基础地址。
- 登录地址。
- 设计态应用管理地址。
- 运行态门户地址。
- 运行态应用中心地址。
- 登录账号。
- 登录密码。

密码仅允许加密保存，不允许明文写入配置文件、日志、文档或 Git。

## 3. 登录策略

1. 后端读取加密凭据。
2. 调用云枢登录接口或通过 Playwright 登录。
3. 获取 Token/Cookie。
4. Token/Cookie 只进入加密凭据表或受控内存缓存。
5. 到期自动刷新或重新登录。
6. 日志和审计中只记录脱敏摘要。

## 4. 设计态元数据同步

需要同步以下内容：

- 应用列表。
- 应用目录。
- 业务模型。
- 数据项字段。
- 表单。
- 表单字段。
- 关联表单字段和实体关系。
- 视图。
- 视图操作按钮。
- 表单设计中的操作按钮。
- 流程。
- 业务规则。
- Action。
- 报表入口或报表配置。

同步结果标准化后写入 MySQL，用于构建应用级知识图谱、检索语料和执行规划。

## 5. 应用级知识图谱映射

- 云枢应用 -> `cloudpivot_apps`。
- 业务模型 -> 数据实体。
- 业务模型数据项 -> 实体字段。
- 关联表单字段 -> 实体关系边。
- 业务模型视图中的操作按钮 -> 视图操作。
- 应用设计中表单设计的操作按钮 -> 表单操作。
- 流程和 Action -> 可执行能力节点。

Agent 必须使用图谱结构校验候选：字段必须属于实体，操作必须属于对应视图或表单，跨实体查询必须存在关系路径。

## 6. 元数据同步流水线

```text
CloudPivot Metadata Sync
  -> Raw JSON 保存到 MySQL
  -> 标准化为应用/实体/字段/关系/表单/按钮
  -> 构建应用级知识图谱
  -> 生成 search documents
  -> 写入 MySQL Metadata Index
  -> 启用向量检索时写入 PostgreSQL pgvector
  -> 后续可选写入 Elasticsearch/OpenSearch
  -> 标记 indexed_at / synced_at
```

要求：

- MySQL 是权威主库。
- PostgreSQL pgvector 是当前向量语义召回索引，必须能从 MySQL `metadata_search_documents` 重建，不能作为权威元数据来源。
- Elasticsearch/OpenSearch 是后续可重建索引，当前阶段不作为核心依赖。
- Milvus 是后续大规模向量检索可选替换或扩展方案。
- 同步失败不能污染已生效索引，应使用批次号或版本号切换。
- 向量索引写入失败不能导致云枢元数据初始化失败；系统必须降级为确定性检索。

## 7. 运行态数据操作

MVP 优先支持：

- 列表查询。
- 详情查询。
- 条件过滤。
- 分页。

当前运行态读取策略：

- 计数类问题只需要运行态总数，使用轻量分页，避免无意义拉取大量记录。
- 列表类问题默认返回少量记录摘要，当前只取 10 条且不补运行态详情，避免普通“再看一下列表”被详情接口拖慢。
- 省份/区域、新老客户、阶段/状态等明确维度追问优先读取轻量列表样本，当前默认 100 条；字段缺失时快速说明缺字段，不进入无界全量扫描。
- 阶段/状态维度若列表字段不可见，只允许补极小详情样本用于判断，当前最多 2 条；回答必须标注样本依据，不能伪称全量分布。
- 分析类问题不再默认进入大模型泛分析或全量详情补取；只有按年客户量这类明确全量分布问题保留 `pageSize=200`、最多 20,000 条记录的全量聚合策略。
- 非显式详情模式下，Connector 不得因为 pageSize 较小而自动逐条调用详情接口；只有 `enrichAllDetails=true` 且数量受控时才补详情。
- 如果真实数据超过保护上限，或当前只拿到样本，回答必须说明当前分析基于上限内数据或样本数据，不能伪称全量。
后续扩展：

- 新增。
- 修改。
- 删除。
- 视图 Action。
- 表单操作。
- 报表。
- 流程提交。
- 附件上传。

写操作、删除、流程、Action、附件上传必须经过确认机制。

### 7.1 数据源真实性要求

- 运行态查询必须使用真实云枢 `schemaCode`，该编码必须来自最近一次成功同步的真实 Metadata Index。
- 如果运行态查询来源为 `local-fallback`，Agent 必须停止生成业务数量、趋势、分析结论或下一步建议。
- `system_opportunity`、`system_customer` 等本地演示编码不得作为真实云枢对象返回给用户。
- 在没有真实元数据、真实运行态权限或接口查询失败时，系统应说明当前缺失项，并引导用户重新配置云枢连接、同步元数据或补充应用/表单名称。
- 审计记录必须保留匹配对象、真实 `schemaCode`、运行态来源、返回总数、返回样本数和脱敏错误摘要，方便排查对象串线或数据来源错误。
- 本轮真实环境验收结果：元数据同步 29 个应用、922 个模型、951 条搜索文档；“系统有多少商机？”命中 `int_bu_oppor` total=237；“系统有多少个客户？”命中 `crm_customer` total=8694；“每年的客户量情况怎么样？”命中 `crm_customer` total=8694、returned=8694 后执行年度聚合。

## 8. API 探测方式

在没有完整接口文档时，通过 Playwright 打开页面并观察网络请求，沉淀为稳定 API Connector。

探测结果不直接成为生产能力，必须经过：

1. 参数脱敏。
2. 权限边界确认。
3. 错误处理设计。
4. 审计字段补充。
5. 测试用例覆盖。

## 9. 浏览器自动化兜底

浏览器自动化只在以下情况使用：

- API 未知或不可用。
- 前端页面存在复杂业务逻辑。
- 需要真实用户页面操作验证。
- 需要截图辅助诊断。
- 附件上传或复杂表单暂未沉淀稳定 API。

浏览器兜底仍必须遵守权限、确认、脱敏和审计策略。

## 10. 权限原则

- 不绕过云枢权限体系。
- 所有请求使用配置账号或当前会话对应账号。
- “我的数据”必须以云枢当前用户权限和字段规则为准。
- 如果使用管理员账号同步元数据，运行态操作仍应按业务用户上下文控制。

## 11. 失败处理

- 登录失败：提示用户检查设置。
- 元数据同步失败：保留上次快照并提示失败原因。
- 查询失败：展示应用、实体、过滤条件和脱敏错误摘要。
- API 失败：判断是否可尝试浏览器兜底。
- 浏览器失败：返回截图和步骤摘要供排查，敏感信息必须脱敏。
