# CPClaw 需求说明

> 本文是产品需求详细设计。整体产品设计思路见 `../00-product-blueprint.md`。

## 1. 背景

CPClaw 是面向云枢低代码平台的超级智能体助手，目标是让用户通过自然语言对话访问和操作云枢应用，体验接近真实员工在浏览器中使用云枢系统。

典型场景：用户输入“查询我的销售订单数据”，CPClaw 需要理解用户意图，从本地已同步的云枢元数据中定位 CRM 应用、销售订单实体、字段和查询能力，按云枢当前账号权限查询数据，返回摘要、表格和执行说明。

产品期望不是简单关键词回复，而是足够智能地理解用户想完成的业务任务。用户可以用自然语言表达查询、分析、统计、新增、修改、流程提交、附件填单等目标；系统需要先判断业务意图和目标对象，结合本地元数据、会话上下文和可用工具决定下一步。若信息不足，系统应主动追问和给出候选方向，而不是只回复“没有理解”。

## 2. 云枢平台理解

云枢低代码平台包含设计态和运行态：

- 设计态：应用开发与配置入口，可获取应用、目录、业务模型、表单、流程、字段说明、业务规则、视图操作按钮和表单操作按钮。
- 运行态：已发布应用的使用入口，用户在此查询、新增、修改、删除数据，查看报表、发起流程和执行业务 Action。

CPClaw 不在用户请求时实时连接云枢做应用能力检索。云枢应用元数据必须先同步到本地库，再基于本地 Metadata Index 和应用级知识图谱做意图匹配、能力定位和执行规划。

## 3. 产品目标

- 提供统一对话式入口，降低业务用户访问云枢应用的门槛。
- 让智能体基于云枢设计态元数据理解应用、实体、字段、关系和操作能力。
- 当前阶段通过本地 Metadata Index 的精确名称、别名、编码、应用路径、全文关键词和复合业务词等非向量检索，快速找到用户需要操作的应用功能项；语义向量检索保留为后续增强。
- 支持多轮对话、新建会话、历史会话保存和查看。
- 支持单轮多应用、多实体、多步骤的任务拆解。
- 支持附件上传和附件驱动填单，例如发票报销、合同录入、Excel 明细导入。
- 支持普通用户配置自己的云枢访问账号和密码、模型 API，用于对话式操作云枢应用。
- 支持管理员配置自己的云枢账号和密码，读取目标云枢环境元数据并在本地初始化构建应用级知识图谱和检索索引。
- 通过 API 优先、浏览器自动化兜底的方式执行用户授权的业务操作。
- 对高风险操作提供预览、确认、审计和脱敏。
- 当用户提出“分析系统中的商机信息”这类表达时，系统应理解为查询商机数据并进行业务分析，而不是要求用户重新说明云枢对象。
- 当用户提出“系统有多少商机？”、“商机一共有几条？”这类明确统计表达时，系统应识别为查询/计数意图，直接匹配商机对象并返回数量，不应再要求用户补充应用或表单。
- 当用户表达不完整或意图不清楚时，系统应说明已理解到的内容，列出缺失信息和可选对象，引导用户补充应用/表单、动作类型、筛选条件或记录引用。

## 4. MVP 功能范围

### 4.1 对话式 UI

- 多轮对话。
- 新建会话。
- 历史会话保存和查看。
- Markdown 消息展示。
- 候选应用/功能推荐。
- 执行计划预览。
- 高风险操作确认卡片。
- 模型选择和思考模式开关。
- 附件上传入口。

### 4.2 系统设置

- 云枢基础地址、登录地址、设计态入口、运行态入口配置。
- 云枢账号配置。
- 云枢密码加密保存，禁止明文入库、入日志、入 Git。
- 模型 API 地址、API Key、模型名称配置。
- 模型能力配置：是否支持思考模式、默认是否启用思考模式。
- 检索中间件连接配置。
- 连接测试和配置有效性检查。

### 4.3 云枢架构初始化

- 获取应用、目录、业务模型、表单、流程、字段、视图操作按钮、表单操作按钮和 Action。
- 将元数据标准化后保存到 MySQL。
- 建立应用级知识图谱。
- 生成检索语料并写入 Elasticsearch/OpenSearch。
- 后续可扩展 Milvus 做大规模向量检索。

### 4.4 智能体执行

- 采用结构化意图理解、槽位抽取、本地检索、执行计划、工具调用和 Reflection。
- 支持查询、分析、统计、创建、修改、删除、流程、Action、附件填单等业务意图识别。
- 查询类场景优先落地。
- 查询/统计类表达只要业务对象和动作足够明确，应直接执行查询；只有缺少对象、动作或必要条件且可能查错时才进入澄清。
- 分析类场景应先查询运行态业务数据，再调用模型或规则生成结论、关键发现、风险信号和下一步建议。
- 写操作、删除、流程提交、Action、附件上传必须二次确认。
- 支持多轮上下文引用。
- 支持用户输入不完整时追问或推荐候选功能；低置信度时不得猜测执行。

### 4.5 云枢访问

- API 优先。
- Playwright Java 浏览器自动化兜底。
- 云枢实时连接只用于连接测试、元数据同步和经确认的运行态操作。

## 5. 非 MVP 范围

- 完整多租户权限体系。
- 企业级统一身份认证和细粒度角色中心。
- 全量流程审批复杂编排。
- 报表大屏和复杂 BI。
- 插件市场。
- 大规模异步任务调度平台。
- 全量生产级浏览器自动化适配。

## 6. 安全要求

- 不保存、不输出用户在对话中提供的任何明文密码。
- GitHub、云枢、模型 API Key、数据库密码、检索中间件密码等凭据必须通过环境变量、系统设置加密存储或安全密钥管理传入。
- 删除、批量修改、流程提交、不可逆 Action、附件上传等高风险操作必须用户确认。
- 所有执行记录、工具输入输出、模型调用摘要和错误信息需脱敏。
- 不把密码、Token、Cookie、API Key、附件敏感原文写入 Prompt。
- 不绕过云枢现有权限体系。
