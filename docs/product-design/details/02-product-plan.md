# CPClaw 产品规划详细设计

> 本文是产品规划详细设计。整体产品设计思路见 `../00-product-blueprint.md`。

## 1. 产品定位

CPClaw 是云枢低代码平台的对话式超级智能体，通过自然语言帮助用户访问和操作云枢应用。

它不是简单聊天机器人，而是具备以下能力的业务操作助手：

- 理解用户业务意图。
- 理解本地已同步的云枢应用知识图谱。
- 根据权限、上下文、记忆和检索结果选择正确应用、实体、字段、关系和 Action。
- 调用 API 或模拟浏览器完成用户确认后的操作。
- 对执行结果进行总结、解释、追问和推荐。

## 2. 用户角色

### 2.1 业务用户

- 查询自己的业务数据。
- 汇总订单、客户、合同、报销、流程等数据。
- 根据对话新增或修改业务记录。
- 上传发票、图片、PDF、Excel 等附件并自动填单。
- 查看历史会话和执行步骤。

### 2.2 实施/管理员

- 配置云枢连接信息。
- 配置模型 API 地址、API Key、模型名称和思考模式能力。
- 配置检索中间件连接。
- 同步云枢应用架构。
- 检查应用能力索引。
- 观察智能体执行日志和审计记录。
- 管理业务别名、纠错记忆和高风险能力开放范围。

## 3. 核心用户旅程

### 3.1 查询数据

1. 用户输入“查询我的销售订单数据”。
2. 系统识别为查询意图。
3. 系统从本地元数据知识图谱和检索索引中匹配应用、实体和字段。
4. 系统根据当前账号权限、“我的”语义和上下文生成查询条件。
5. 系统调用云枢运行态 API 查询数据。
6. 系统返回自然语言摘要、Markdown 表格、查询条件和匹配原因。

### 3.2 汇总分析

1. 用户提出统计、排行、趋势或对比需求。
2. 系统定位指标字段、维度字段和时间范围。
3. 如信息不足，系统追问或推荐可选指标。
4. 系统执行聚合查询或拉取后汇总。
5. 系统返回结论、明细依据和下一步建议。

### 3.3 新增数据

1. 用户输入“帮我新增一个客户”。
2. 系统匹配客户实体、表单和必填字段。
3. 如果信息不足，系统追问。
4. 系统生成待提交数据预览。
5. 用户确认后提交。
6. 系统返回新增结果和记录链接。

### 3.4 修改/删除数据

1. 用户描述要修改或删除的数据。
2. 系统先定位目标记录。
3. 系统展示修改前后差异或删除影响范围。
4. 用户确认后执行。
5. 系统返回执行结果和审计摘要。

### 3.5 多轮上下文操作

1. 用户先查询“我的商机”。
2. 系统返回候选记录并保存可引用对象。
3. 用户说“给第一条写一条跟进”。
4. 系统解析“第一条”对应上一轮记录。
5. 系统追问缺失字段或生成写入预览。
6. 用户确认后执行。

### 3.6 单轮多应用/多实体操作

1. 用户输入“查出我的重点客户，把相关商机列出来，并给本周要跟进的商机创建待办”。
2. 系统拆成查询客户、查询商机、筛选本周跟进、创建待办四类节点。
3. 查询节点自动执行。
4. 创建待办节点展示确认摘要。
5. 用户确认后执行写操作。

### 3.7 附件驱动填单

1. 用户上传发票并描述“帮我填写报销单”。
2. 系统解析附件内容，抽取金额、日期、发票号、供应商等信息。
3. 系统匹配报销应用、报销单实体和表单字段。
4. 系统展示字段映射、缺失字段和附件上传预览。
5. 用户确认后创建报销单并上传附件。

## 4. MVP 页面

### 4.1 对话页

- 左侧历史会话列表。
- 主区域消息流。
- Markdown 消息展示。
- 模型选择器。
- 思考模式开关。
- 附件上传入口。
- 候选推荐卡片。
- 执行计划预览卡片。
- 高风险确认卡片。
- 执行状态和步骤时间线。

### 4.2 设置页

- 云枢基础地址。
- 设计态入口。
- 运行态入口。
- 登录账号。
- 登录密码输入框，仅写入，不回显。
- 模型配置列表。
- 模型 API 地址、API Key、模型名称。
- 模型能力：是否支持思考模式、默认思考模式。
- 检索中间件配置。
- 连接测试。
- 同步云枢架构按钮。

### 4.3 架构页

- 应用列表。
- 实体数量。
- 字段数量。
- 关系数量。
- 视图操作和表单操作数量。
- 最后同步时间。
- 索引构建状态。

### 4.4 审计页

MVP 可简化为后端接口，后续提供页面：

- Agent 执行记录。
- 工具调用记录。
- 确认记录。
- 脱敏错误摘要。

## 5. 交互模式

- 澄清模式：缺少必要信息时，要求用户补充。
- 候选推荐模式：存在多个相近应用或功能时，提供候选表格和匹配原因。
- 执行计划模式：执行前展示“我理解你要做什么、将操作哪里、使用什么条件”。
- 风险确认模式：写操作和高风险操作必须确认。
- 结果解释模式：返回摘要、表格、条件、数据来源、执行步骤和下一步建议。

## 6. 产品原则

- 先查询，后写入。
- 不确定就追问，不猜测执行。
- 本地元数据检索优先，不实时连接云枢做匹配。
- API 优先，浏览器兜底。
- 用户确认优先于自动执行。
- 对业务结果负责，对执行路径透明。
- 记忆只增强匹配准确率，不能绕过权限和确认。
