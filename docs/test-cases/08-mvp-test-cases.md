# MVP 测试用例

## TC-001 前端启动

- 前置条件：已安装前端依赖。
- 步骤：
  1. 进入 `web/`。
  2. 启动 Vue 3 开发服务。
  3. 打开前端页面。
- 预期结果：
  - 页面可访问。
  - 无启动错误。
  - 可看到聊天入口、会话列表入口和设置入口。

## TC-002 后端启动

- 前置条件：已配置 Java 21 和后端依赖。
- 步骤：
  1. 进入 `server/`。
  2. 启动 Spring Boot 应用。
  3. 调用健康检查接口。
- 预期结果：
  - Spring Boot 应用启动成功。
  - 健康检查返回成功状态。
  - 日志中不出现明文敏感信息。

## TC-003 MySQL 连接

- 前置条件：MySQL 可访问，数据库连接参数通过环境变量提供。
- 步骤：
  1. 启动后端。
  2. 执行数据库连接检查。
  3. 执行 Flyway 迁移。
- 预期结果：
  - 后端可连接 MySQL。
  - 迁移执行成功。
  - 必要表结构创建成功。

## TC-004 系统设置保存

- 前置条件：前后端均已启动。
- 步骤：
  1. 打开设置页。
  2. 填写云枢访问地址和账号。
  3. 填写密码并保存。
  4. 重新打开设置页。
- 预期结果：
  - 设置保存成功。
  - 前端不回显密码。
  - 设置查询接口不返回密码或密文内容。
  - 数据库中只保存加密凭据，不保存明文密码。

## TC-005 会话保存与读取

- 前置条件：前后端均已启动，MySQL 连接正常。
- 步骤：
  1. 新建会话。
  2. 输入一条用户消息。
  3. 等待系统生成回复。
  4. 刷新页面并查看历史会话。
- 预期结果：
  - 会话创建成功。
  - 用户消息和助手消息均保存成功。
  - 历史会话可查看。
  - 会话详情可恢复消息列表。

## TC-006 查询类 Agent 模拟链路

- 前置条件：已存在模拟元数据或已完成云枢架构同步，且本地 Metadata Index 可用。
- 步骤：
  1. 新建会话。
  2. 选择系统设置中已配置的模型。
  3. 如果模型支持思考模式，启用或关闭思考模式。
  4. 输入“查询我的销售订单数据”。
  5. 观察 Agent 执行步骤。
- 预期结果：
  - 意图识别为 `query_data`。
  - 能力匹配只查询本地 Metadata Index，不实时连接云枢检索。
  - 能力匹配到 CRM 应用和销售订单实体。
  - 生成查询计划。
  - 返回自然语言摘要、Markdown 表格、查询条件和匹配原因。
  - 工具输入输出审计记录已脱敏。

## TC-007 云枢连接测试

- 前置条件：已在设置页配置云枢地址和账号凭据。
- 步骤：
  1. 点击“测试连接”。
  2. 后端发起云枢登录或连接检查。
- 预期结果：
  - 连接成功时返回成功状态。
  - 连接失败时返回脱敏错误信息。
  - 日志不出现密码、Token、Cookie。

## TC-008 云枢架构同步

- 前置条件：云枢连接测试成功。
- 步骤：
  1. 点击“同步云枢架构”。
  2. 后端拉取应用、模型、表单、字段和 Action 元数据。
  3. 查看同步结果。
- 预期结果：
  - 元数据保存到 MySQL。
  - 常用字段被结构化索引。
  - 原始响应保存到 `raw_json`。
  - 页面显示同步时间和同步结果摘要。

## TC-009 高风险操作确认

- 前置条件：Agent 已能识别写操作意图。
- 步骤：
  1. 输入删除、批量修改、流程提交或触发 Action 类请求。
  2. 观察系统响应。
- 预期结果：
  - 系统不直接执行高风险操作。
  - 系统展示待执行操作摘要。
  - 系统要求用户二次确认。
  - 未确认前不调用实际写操作工具。

## TC-010 敏感信息脱敏检查

- 前置条件：已完成一次设置保存、连接测试或 Agent 工具调用。
- 步骤：
  1. 检查后端日志。
  2. 检查工具调用审计记录。
  3. 检查模型调用摘要。
  4. 检查接口响应。
  5. 检查 Git diff。
- 预期结果：
  - 不存在明文密码、Token、Cookie、Session ID、API Key。
  - 敏感字段已按规则脱敏。
  - `.env`、本地配置、密钥文件不在 Git diff 中。

## TC-011 模型配置与思考模式

- 前置条件：前后端均已启动。
- 步骤：
  1. 打开设置页。
  2. 新增一个 OpenAI 兼容模型配置。
  3. 填写 API 地址、API Key、模型名称和是否支持思考模式。
  4. 保存并测试模型连接。
  5. 回到对话页选择该模型。
- 预期结果：
  - 模型配置保存成功。
  - API Key 不回显，接口不返回明文或密文。
  - 对话页可选择该模型。
  - 仅当模型支持思考模式时展示思考模式开关。

## TC-012 附件驱动填单链路

- 前置条件：已存在报销类应用元数据或模拟元数据。
- 步骤：
  1. 新建会话。
  2. 上传发票图片或 PDF。
  3. 输入“帮我填写报销单”。
  4. 查看字段映射和缺失字段提示。
- 预期结果：
  - 附件上传成功。
  - 系统解析附件并生成结构化结果。
  - 系统匹配报销应用、实体和表单字段。
  - 系统展示字段映射和附件上传预览。
  - 用户确认前不创建云枢记录、不上传附件到云枢。

## TC-013 多轮上下文操作

- 前置条件：已存在查询类模拟数据。
- 步骤：
  1. 输入“找到我的商机信息”。
  2. 系统返回商机列表。
  3. 输入“给第一条写一条跟进记录”。
- 预期结果：
  - 系统能将“第一条”解析为上一轮第一条商机。
  - 系统展示待写入跟进内容或追问缺失内容。
  - 用户确认前不调用写操作工具。

## TC-014 单轮多应用/多实体计划

- 前置条件：已存在客户、商机和待办相关模拟元数据。
- 步骤：
  1. 输入“查出我的重点客户，把相关商机列出来，并给本周要跟进的商机创建待办”。
  2. 查看系统生成的执行计划。
- 预期结果：
  - 系统拆解为多个 DAG 节点。
  - 查询节点可先执行。
  - 创建待办节点被识别为写操作并要求确认。
  - 计划中展示跨实体关系路径和风险说明。

## TC-015 分析类自然语言意图

- 前置条件：已存在商机相关模拟元数据或已完成云枢架构同步，且普通用户云枢账号可查询商机运行态数据。
- 步骤：
  1. 新建会话。
  2. 输入“分析系统中的商机信息”。
  3. 查看 Agent 识别结果、候选对象和回答内容。
- 预期结果：
  - 意图识别为 `analyze_data`。
  - 系统匹配商机相关应用/实体，不要求用户重新说明云枢对象。
  - 系统先查询云枢运行态商机数据。
  - 系统返回分析型回答，至少包含结论摘要、关键发现、风险信号和下一步建议。
  - 查询和模型分析相关审计记录已脱敏。

## TC-016 模糊意图澄清

- 前置条件：已完成云枢元数据初始化或存在模拟元数据。
- 步骤：
  1. 新建会话。
  2. 输入“帮我处理一下”或其它缺少对象和动作的模糊请求。
  3. 查看系统回复。
- 预期结果：
  - 意图识别为 `clarify_intent`。
  - 系统不执行云枢工具，不修改数据，不触发高风险确认。
  - 系统说明已理解到的内容和当前缺失的信息。
  - 系统引导用户补充应用/表单、动作类型、筛选条件或记录引用。
  - 如果本地 Metadata Index 有候选对象，系统展示部分候选对象供用户选择。

## TC-017 明确商机数量查询

- 前置条件：已存在商机相关模拟元数据或已完成云枢架构同步，且普通用户云枢账号可查询商机运行态数据。
- 步骤：
  1. 新建会话。
  2. 输入“系统有多少商机？”。
  3. 查看 Agent 识别结果、候选对象和回答内容。
- 预期结果：
  - 意图识别为 `query_data`。
  - 系统将“有多少、多少、？”等识别为计数表达，不把它们当成业务对象关键词。
  - 系统匹配商机相关应用/实体，不要求用户重新说明应用或表单。
  - 系统调用云枢运行态查询商机数量。
  - 系统直接返回总数，不进入 `clarify_intent`，不触发高风险确认。

## TC-018 ReAct + Reflection 执行链与审计

- 前置条件：已存在商机相关模拟元数据或已完成云枢架构同步，且普通用户云枢账号可查询商机运行态数据。
- 步骤：
  1. 新建会话。
  2. 输入“系统有多少商机？”。
  3. 查看 Agent 响应步骤。
  4. 打开审计页或调用审计接口查看本次 Agent Run。
  5. 在同一会话中输入“给第一条商机写一条跟进记录”。
- 预期结果：
  - 查询响应步骤包含 `Observe 观察上下文`、`Think 理解意图`、`Act 执行动作`、`Reflect 反思检查`。
  - 审计 `plan_json` 包含 `mode=react-reflection-mvp`、观察结果、思考结果和计划动作。
  - 审计 `reflection_json` 包含反思状态、检查项、是否通过和是否需要用户输入。
  - 明确查询类请求反思结果为已完成或可返回结果。
  - 高风险写操作必须生成确认，反思状态为等待确认。
  - 缺少必要槽位时，例如没有提供跟进内容，系统进入 `clarify_intent`，不直接生成写操作确认，不执行云枢写操作。

## TC-020 按年客户量分析

- 前置条件：已存在客户相关模拟元数据或已完成云枢架构同步，且客户运行态数据中存在创建时间、登记时间、年份等可识别时间字段。
- 步骤：
  1. 新建会话。
  2. 输入“每年的客户量情况怎么样？”。
  3. 查看 Agent 识别结果、候选对象、执行步骤和回答内容。
- 预期结果：
  - 意图识别为 `analyze_data`。
  - 系统识别动作是分析，业务对象是客户，分析维度是年份。
  - 系统匹配客户相关应用/实体，不要求用户重新说明云枢对象。
  - 系统先查询客户运行态数据；分析类请求应尽量分页拉取完整数据后再聚合。
  - 系统按年份聚合客户量，返回年度分布、峰值年份、趋势判断和下一步下钻建议。
  - 响应步骤包含 `Observe 观察上下文`、`Think 理解意图`、`Act 执行动作`、`Reflect 反思检查`。
  - 如果无法识别年份字段，系统说明缺失字段并给出补充方向，不编造年度分布。
