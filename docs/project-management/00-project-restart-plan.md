## 2. 项目理解

CPClaw 的核心目标是让用户通过自然语言访问和操作云枢应用。系统需要先由管理员同步云枢设计态元数据到本地 MySQL，形成应用级知识图谱和 Metadata Index；普通用户在对话中提出查询或操作目标，Agent 基于本地元数据匹配应用、实体、字段和能力，再调用云枢运行态 API 或生成确认卡片。

当前 MVP 的核心闭环是：

1. 用户配置云枢账号和模型配置。
2. 管理员初始化云枢元数据。
3. 用户在对话页输入查询或操作目标。
4. Agent 识别意图，检索本地元数据，生成计划摘要。
5. 查询类请求调用云枢运行态查询接口。
6. 写操作、删除、流程、Action 和附件上传先进入风险确认。
7. 系统写入 Agent run、工具调用和确认记录，供审计查询。

## 3. 当前代码基线

### 3.1 后端

- 技术栈：Java 21、Spring Boot 3、Spring Web MVC、Spring Data JPA、Flyway、MySQL、H2 测试库。
- 已有模块：设置、凭据加密、模型配置、会话、消息、元数据、搜索、Agent、云枢连接器、运行态查询、附件上传、审计、确认记录。
- 云枢能力：已实现连接测试、元数据同步探测、运行态列表查询接口调用。
- Agent 能力：当前为规则型 MVP，实现关键词意图识别、本地 Metadata Index 匹配、查询执行、风险确认生成和审计记录。
- 附件能力：已实现上传、存储、SHA-256 计算和状态记录，尚未实现解析和字段映射。
- 模型能力：模型配置已入库，OpenAI 兼容模型网关仍是占位实现。

### 3.2 前端

- 技术栈：Vue 3、TypeScript、Vite、Pinia、Vue Router、Element Plus。
- 已有页面：对话页、云枢账号设置页、元数据页、审计页。
- 当前可见路由：对话页和云枢账号设置页。
- 重要缺口：元数据页和审计页组件存在，但当前路由和侧边导航未暴露；设置页当前只覆盖普通用户云枢账号，管理员元数据设置和模型配置需要恢复到页面体验。

### 3.3 文档与测试

- 已有产品文档：产品蓝图、需求说明、产品规划详细设计。
- 已有技术文档：技术蓝图、系统架构、Agent、云枢集成、数据模型、安全设计。
- 已有测试文档：测试计划、MVP 测试用例、MVP 验证执行记录。
- 2026-06-18 复测结果：后端 `mvn test` 通过，前端 `npm run build` 通过。
- 前端构建警告：Rollup 依赖注释警告和 chunk 超过 500 kB 警告，不阻断构建。
