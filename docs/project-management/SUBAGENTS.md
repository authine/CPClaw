当前重点不是新增大功能，而是把 Agent 的主线拉回真实云枢数据：用户自然语言意图必须先映射到已同步的真实云枢元数据，再用真实 `schemaCode` 调用运行态接口，最后基于真实返回数据生成统计或大模型分析。团队已完成本轮关键纠偏并通过真实云枢复测：移除 fallback 中的“系统商机/系统客户”演示业务模型和演示运行态数据；运行态服务遇到 `local-fallback` 会拒绝生成业务答案；元数据检索改为业务词全量候选召回后统一排序；Agent 的 `Think` 阶段展示动作、业务对象、分析维度、筛选、候选对象和候选编码；QA 新增真实 schema mock 用例和 fallback guard 用例，确保 `system_opportunity`、`system_customer` 不能再冒充真实云枢编码。真实云枢复测已同步 29 个应用、922 个模型、951 条搜索文档；商机命中 `int_bu_oppor`，客户命中 `crm_customer`。本轮已新增对象级多轮上下文继承：追问“分别在什么阶段？”会继承上一轮真实运行态对象和 `schemaCode` 并重新查询运行态数据；真实代理复测返回商机阶段分布未签约 135、暂停 13、已签约 65、停止 20、丢单 4；同名对象场景不会跳回默认对象。剩余主要风险是生产级 DAG 多工具循环、结构化意图、字段级图谱、写操作执行、记录级上下文 ID 引用和阶段/状态分布性能优化仍待增强。

## 角色工作记录

### 产品经理

- 已确认项目目标：面向云枢低代码平台的对话式超级智能体。
- 已确认 MVP 范围：对话入口、历史会话、系统设置、元数据初始化、查询类场景、风险操作二次确认和审计脱敏。
- 当前关注点：先保证已有功能可运行、可保存、可验证；分析类需求必须按“先查业务数据，再由模型/规则分析”处理；不确定时必须先澄清，而不是直接判定无法理解。
- 本轮新增口径：明确统计查询必须直接执行，但前提是命中真实云枢元数据中的模型；例如“系统有多少商机？”动作是统计数量、对象是商机，应查真实商机 `schemaCode`，不能用本地演示编码回答。
- 本轮新增口径：客户类表达也必须按业务语义执行，例如“系统有多少个客户？”是统计客户数量，“每年的客户量情况怎么样？”是按年份分析客户量；同样必须基于真实客户运行态数据。
- 本轮新增口径：读数据回答必须展示可审计执行过程，例如用户问商机或客户时，应能看到动作、对象、维度、候选编码、真实 `schemaCode`、原始摘要和结论生成方式；fallback 来源必须阻断业务答案。
- 本轮 QA 结论：真实数据主线保护已通过自动化验证和真实云枢复测；完整多轮上下文记录引用、模型驱动 DAG 和多工具循环仍需后续规划。
