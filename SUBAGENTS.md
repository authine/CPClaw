# CPClaw SubAgent 协作看板

本文件用于维护当前项目的 SubAgent / 协作角色状态。由于 Codex 客户端右侧面板不一定展示真实 SubAgent 列表，本仓库使用这个文件作为稳定的团队看板，并在关键任务后同步更新。

## 当前结论

**SubAgent 看板已建立，本轮团队测试已完成。**

当前重点不是新增大功能，而是把已有前端工作台、账号与模型配置、元数据初始化、审计入口和后端服务联调跑通，并持续修正 Agent 的自然语言理解体验。本轮由 QA 测试工程师牵头，已完成前端构建、后端自动化测试、临时 H2 后端、设置保存、元数据同步和对话接口检查；针对用户反馈“系统有多少商机？”明确统计意图被误澄清，已补计数表达清洗、统计意图识别和回归测试。随后团队完成 ReAct + Reflection MVP 改造：后端每轮对话经过 Observe、Think、Plan、Act、Reflect，响应步骤和审计均可追踪；本轮继续补齐“系统有多少个客户？”与“每年的客户量情况怎么样？”两个自然语言场景，新增系统客户 fallback 元数据和按年分析能力。QA 当前专项验证覆盖 14 条自然语言用例，其中 13 条严格验收全部通过，1 条复杂上下文用例验证安全澄清能力。剩余主要风险是真实 MySQL 持久化环境尚未启动验证，以及生产级 DAG 多工具循环、结构化意图和完整上下文记录引用仍待增强。

## 角色工作记录

### 产品经理

- 已确认项目目标：面向云枢低代码平台的对话式超级智能体。
- 已确认 MVP 范围：对话入口、历史会话、系统设置、元数据初始化、查询类场景、风险操作二次确认和审计脱敏。
- 当前关注点：先保证已有功能可运行、可保存、可验证；分析类需求必须按“先查业务数据，再由模型/规则分析”处理；不确定时必须先澄清，而不是直接判定无法理解。
- 本轮新增口径：明确统计查询必须直接执行，例如“系统有多少商机？”的动作是统计数量、对象是商机，不应再追问应用或表单。
- 本轮新增口径：客户类表达也必须按业务语义执行，例如“系统有多少个客户？”是统计客户数量，“每年的客户量情况怎么样？”是按年份分析客户量，不应再追问应用或表单。
- 本轮 QA 结论：智能体 MVP 可用于用户继续验证明确查询、统计、分析、按年维度分析、澄清、写操作确认和 ReAct + Reflection 执行过程；完整多轮上下文记录引用、模型驱动 DAG 和多工具循环仍需后续规划。
- 已同步文档：产品需求和产品规划已补充“理解意图、必要时澄清、执行任务、解释结果”的产品基线。
