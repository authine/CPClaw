# CPClaw Figma UI Prototype Delivery

项目：云枢超级智能体 CPClaw

本目录包含两类交付物：

- `prototype.html`：本地可点击原型，第一屏就是主对话工作台。
- `figma-plugin/`：Figma 插件生成器，可在当前 Figma 文件中一键创建 CPClaw 项目页面、主流程原型、组件规范、状态页和窄屏适配方案。

## Figma 连接状态

注意：Figma 官方 REST API 主要用于读取文件、评论、变量和资源，不提供创建新 Figma 文件或绘制画布节点的通用接口。因此“创建完整 Figma 原型”需要在 Figma Desktop 或 Figma Web 的画布环境中运行插件生成器。

## 在 Figma 中生成 CPClaw 项目

1. 打开 Figma，创建一个新的 Design 文件，命名为 `CPClaw - 对话式超级智能体工作台`。
2. 进入 `Plugins -> Development -> Import plugin from manifest...`。
3. 选择本目录中的 `figma-plugin/manifest.json`。
4. 运行 `Plugins -> Development -> CPClaw UI Prototype Generator`。
5. 插件会生成以下页面：
   - `01 主流程原型`
   - `02 组件规范`
   - `03 主要状态`
   - `04 窄屏适配`

## 页面覆盖

主对话工作台：
左侧历史会话，中间消息流和输入框，右侧 Agent 执行时间线、元数据匹配、候选选择和审计编号。第一屏不是首页或营销页。

设置面板：
包含普通用户云枢账号配置、管理员元数据环境配置、模型 API 配置、思考模式配置、连接测试状态。密码、Token、API Key 均使用占位符，不展示真实值。

元数据初始化界面：
包含初始化按钮、同步进度、应用列表、实体/字段/Action 统计、检索框和同步失败提示。

审计详情界面：
包含 Agent Run ID 查询、意图、风险等级、状态、执行计划、工具调用记录、输入输出脱敏展示。

高风险确认卡片：
包含操作类型、影响对象、风险等级、系统将执行什么、确认按钮、取消按钮，以及“确认前不会执行真实写入”的明确状态。

组件规范页：
覆盖会话列表项、用户消息气泡、助手消息气泡、Markdown/表格结果、Agent 执行时间线、元数据匹配卡片、候选应用/实体选择器、高风险确认卡片、附件上传入口、设置表单、连接测试结果状态、空/加载/错误等状态。

窄屏适配：
390px 方案隐藏左右栏，将 Agent 过程、元数据匹配和确认卡片转为底部抽屉，保证输入框始终可见。

## 验收流程映射

1. 配置云枢账号：打开设置面板，保存地址、账号、密码，查看连接测试成功状态。
2. 初始化云枢元数据：打开元数据初始化，点击开始同步，查看应用数、实体数、索引文档数和同步时间。
3. 新建会话：点击左侧新建会话。
4. 输入“查询我的销售订单数据”：点击示例按钮或在输入框发送。
5. 查看匹配对象、执行计划和结果：主消息展示意图、命中应用/实体/字段、执行计划、摘要和表格；右侧展示执行时间线和元数据匹配。
6. 输入写操作请求：点击“给这个客户新增跟进记录”。
7. 看到高风险确认卡片：确认卡片展示写入对象和风险，确认前不会执行真实写入。
8. 查看历史会话和审计记录：左侧历史会话可见，审计详情可按 Agent Run ID 查询。

## 视觉原则

- 圆角不超过 8px。
- 企业工作台风格，信息密度适中。
- 避免营销式 hero、大面积装饰图、夸张渐变和卡片堆叠。
- 风险状态区分清楚：低风险绿色，中风险黄色/橙色，高风险红色。
- 所有可审计步骤均显示 Agent Run ID 或审计入口。
